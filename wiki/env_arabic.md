# [لينكس] Bash env الاستخدام: [إدارة المتغيرات البيئية]

## نظرة عامة
يستخدم أمر `env` في Bash لإدارة المتغيرات البيئية، حيث يمكنه عرض المتغيرات الحالية أو تشغيل برنامج مع مجموعة محددة من المتغيرات.

## الاستخدام
الصيغة الأساسية للأمر هي:
```bash
env [options] [arguments]
```

## الخيارات الشائعة
- `-i`: تشغيل الأمر في بيئة فارغة، مما يعني عدم استخدام أي متغيرات بيئية موجودة.
- `-u VARIABLE`: إزالة متغير بيئي محدد قبل تشغيل الأمر.
- `VARIABLE=value`: تعيين قيمة لمتغير بيئي قبل تشغيل الأمر.

## أمثلة شائعة
- عرض جميع المتغيرات البيئية الحالية:
  ```bash
  env
  ```

- تشغيل أمر مع متغير بيئي جديد:
  ```bash
  env MY_VAR=value my_command
  ```

- تشغيل أمر في بيئة فارغة:
  ```bash
  env -i my_command
  ```

- إزالة متغير بيئي قبل تشغيل الأمر:
  ```bash
  env -u MY_VAR my_command
  ```

## نصائح
- استخدم `env` لتجنب مشاكل التكوين عند تشغيل البرامج، خاصة في بيئات متعددة.
- يمكنك استخدام `env` لتجربة إعدادات جديدة دون تغيير المتغيرات البيئية الحالية.
- تأكد من استخدام `-i` بحذر، حيث سيؤدي ذلك إلى فقدان جميع المتغيرات البيئية.
